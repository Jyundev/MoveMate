# MoveMate (무브메이트)

### 도착 이후 이동 선택을 돕는 실시간 공공데이터 기반 이동 전략 서비스

---

# 1. 개발 결과물 개요

> MoveMate는 "도착 이후 무엇을 선택해야 하는지"를 결정해주는 이동 전략 서비스이다.

MoveMate는 공모전 필수 활용 데이터셋인 **공영 물품보관함 현황 실시간 정보**와 **전국 공영자전거 실시간 정보**를 핵심 입력 데이터로 활용하여, 사용자가 주요 거점에 도착한 이후 발생하는 **마지막 이동 구간의 선택 문제를 해결하는 서비스**이다.

본 데모 버전은 서울 주요 3개 거점(서울역, 강남역, 성수역)과 9개 목적지를 대상으로 구현하였으며, 거점 DB 확장을 통해 전국 주요 역사·터미널로 서비스 범위를 확장할 수 있도록 설계하였다.

기존 지도 및 교통 서비스는 출발지에서 목적지까지의 경로를 안내하며 최단 거리 또는 최소 시간 중심의 이동 방법을 제공한다. 그러나 실제 사용자들은 도착 이후 다음과 같은 문제를 겪는다.

- 짐이 있는 상황에서 이동 수단 선택이 어려움
- 도보와 자전거 중 어떤 선택이 더 적절한지 판단 어려움
- 공공자전거 및 보관함의 실시간 이용 가능 여부 확인이 어려움
- 여러 앱을 번갈아 확인해야 하는 정보 탐색 피로 존재

현실에서는 단순히 "갈 수 있는 경로"보다 **"지금 상황에서 어떤 선택이 가장 적절한가"**가 더 중요한 문제이다.

MoveMate는 공영자전거 및 공영 물품보관함 실시간 데이터를 기반으로 사용자의 상황(짐 유무, 도보 선호도 등)을 종합 분석하여 **가장 실행 가능성이 높은 이동 전략을 제안하는 서비스**이다. 단순 추천이 아닌, **사용자의 실제 행동으로 이어지는 의사결정 지원**을 목표로 한다.

---

## 1-1. 사용자 이용 시나리오

서울역에 도착한 사용자 A는 명동으로 이동하려 한다. 짐이 있어 도보와 자전거 중 어떤 선택이 적절한지 판단이 어렵다.

MoveMate는 주변 보관함 여석과 자전거 현황을 실시간으로 분석하여, 짐을 먼저 맡긴 후 이동하는 전략을 추천한다. 사용자는 복잡한 정보 탐색 없이 **즉시 이동 방식을 선택하고 행동으로 옮길 수 있다.**

# 2. 개발 결과물의 기능 및 특징

## 2-1. 서비스 목적 및 배경

도시 이동은 단순한 경로 탐색을 넘어 **도착 이후의 이동 방식 선택까지 포함된 복합적인 의사결정 과정**이다.

특히 주요 거점에 도착한 사용자는 다음과 같은 상황을 반복적으로 경험한다.

- 목적지까지 애매한 거리 (700~1,400m)
- 짐 유무에 따른 이동 전략 변화
- 공공 인프라 실시간 이용 여부 확인 어려움
- 여러 정보 비교의 번거로움

이러한 문제는 단순 경로 안내로 해결되지 않으며, **사용자 상황 + 실시간 데이터 기반 판단 지원 서비스가 필요하다.**

---

## 2-2. 핵심 기능 및 구성

### 기능 1. 이동 전략 추천

사용자가 도착 거점, 최종 목적지, 짐 유무, 도보 선호도를 선택하면 **최대 3개의 이동 전략을 점수 기반으로 추천**한다. 짐이 있을 경우 짐 보관 위치 선호(거점 근처 / 목적지 근처 / 추천)를 추가로 선택할 수 있다.

제공 전략 유형:

| 전략 코드 | 설명 |
|-----------|------|
| WALK | 도보 이동 |
| BIKE | 공영자전거 이용 |
| LOCKER_WALK | 짐 보관 후 도보 이동 |
| LOCKER_BIKE | 짐 보관 후 자전거 이동 |

---

### 기능 2. 실행 가능성 기반 판단

다음 요소를 종합 분석하여 단순히 빠른 경로가 아닌 **실제로 실행 가능한 전략**을 우선 제공한다.

- 자전거 실시간 잔여 수 (거점 반경 1000m 내 — 단거리 이동 수단 특성상 대여소까지 과도한 도보 방지)
- 보관함 실시간 여석 (거점 반경 1500m 내 — 도보로 접근 가능한 현실적 범위)
- Haversine 공식 기반 직선 거리 + 도로 보정계수(1.3x) 적용 이동 시간 추정
- 사용자 조건 (짐 유무, 도보 선호도)

---

### 기능 3. 실패 위험도(failRisk) 분석

각 전략의 실패 가능성을 3개 요소로 정량 평가한다.

| 요소 | 내용 | 최대 점수 |
|------|------|-----------|
| 자원 부족 | 자전거 / 보관함 부족 | 35점 |
| 거리·피로도 | 도보 시간 기준 부담 | 20점 |
| 전략 복잡도 | 필요한 인프라 단계 수 | 5점 |

각 요소는 실제 이동 실패 가능성에 직접적으로 영향을 주는 요인으로, 자원 부족은 이용 불가 상황, 거리·피로도는 이동 지속성 저하, 전략 복잡도는 실행 과정의 이탈 가능성을 반영한다.

합산 점수 30점 미만 → LOW, 30~59점 → MEDIUM, 60점 이상 → HIGH로 제공한다.

---

### 기능 4. AI 기반 자연어 판단 근거 제공

본 서비스는 LLM(OpenAI gpt-4.1-mini)을 활용하여 추천 결과의 핵심 판단 근거를 **1문장(80자 이내)으로** 자동 생성한다.

구조:
- 추천 및 점수 계산: 규칙 기반 알고리즘
- 판단 근거 요약: AI (gpt-4.1-mini, max_output_tokens 60)
- AI 장애 시: 규칙 기반 대체 문장 자동 생성 (서비스 무중단 보장)

이는 Explainable AI 기반으로 추천 결과의 판단 근거를 사용자에게 설명하여 선택에 대한 신뢰도를 높이는 구조이다.

---

### 기능 5. 지도 기반 시각화

- 출발 거점 / 목적지 마커 표시
- 전략별 이동 경로 표시 (도보: 실선, 자전거: 점선 / 위경도 기반 추정 경로)
- 현재 위치 기반 거점까지의 거리 실시간 표시

# 3. 기존 서비스와 차별성 및 독창성

## 3-1. 기존 서비스와의 차별점

| 구분 | 기존 서비스 | MoveMate |
|------|-------------|----------|
| 목적 | 경로 안내 | 이동 전략 선택 지원 |
| 판단 기준 | 시간 / 거리 | 사용자 상황 + 실시간 데이터 |
| 공공데이터 활용 | 참고 수준 | 핵심 판단 근거로 가공 |
| 실패 위험 고려 | 없음 | 있음 (failRisk 3단계) |
| 설명 제공 | 없음 | AI 판단 근거 요약 |

---

## 3-2. 독창성

1. **도착 이후 이동 문제에 집중** — 기존 서비스가 다루지 않는 라스트마일 의사결정 구간을 타깃으로 함
2. **공공데이터 기반 의사결정** — 실시간 자전거·보관함 데이터를 단순 표시가 아닌 전략 판단의 입력값으로 활용
3. **실행 가능성 + 실패 위험도 분리** — 추천과 위험 평가를 별도 지표로 제공하여 사용자 신뢰도 향상
4. **알고리즘과 AI의 역할 분리** — 추천은 결정론적 알고리즘, 설명은 AI로 분리하여 신뢰성과 가독성을 동시에 확보

# 4. 공공데이터 활용성

## 4-1. 활용 공공데이터 목록

### 서울시 공공자전거(따릉이) 실시간 대여 정보

- 제공기관: 서울특별시
- API 명칭: `inf_101_00010002_v2` (서울시 공공자전거 실시간 대여정보)
- 수집 정보: 정류소별 위치(위경도), 자전거 잔여 수, 총 거치대 수
- 호출 방식: REST OpenAPI (JSON)

### 물품보관함 위치·여석 정보

- 제공기관: 행정안전부 (공공데이터포털을 통해 제공)
- API 명칭: `locker_info_v2` (물품보관함 기본 정보), `locker_realtime_use_v2` (실시간 이용 현황)
- 수집 정보: 보관함 위치(위경도), 총 칸 수, 실시간 사용 중 칸 수, 여석
- 호출 방식: REST OpenAPI (JSON)

---

## 4-2. 데이터 처리 방식

공공데이터를 단순 조회·표시하는 것을 넘어, **전략 판단의 입력 데이터로 가공하여 활용**한다.

처리 절차:

1. 사용자 입력(거점 ID) 수신 시 자전거 API, 보관함 info API, 보관함 realtime API를 **병렬 호출**하여 응답 시간 최소화
2. 각 시설물의 위경도와 거점 좌표 간 **Haversine 공식**으로 직선 거리를 계산하여 반경 내 시설만 필터링 (자전거 1000m, 보관함 1500m)
3. 잔여 자원 수를 기반으로 전략별 점수 산정 및 failRisk 계산
4. API 장애 발생 시 해당 전략(BIKE, LOCKER 계열)을 제외하고 가용 전략만 제공하여 **서비스 연속성 유지**

---

## 4-3. 지속성 및 확장성

- 표준 REST OpenAPI 기반으로 지속적인 데이터 활용 가능
- 거점 DB 확장만으로 전국 주요 역사·터미널로 서비스 범위 확장 가능
- 버스 실시간 도착 정보 등 추가 공공데이터 연계 구조 보유

# 5. 기술성과 구현 및 실현 가능성

## 5-1. 주요 기술 스택

| 분류 | 기술 |
|------|------|
| 프레임워크 | Next.js 15 (App Router), React 19 |
| 언어 | TypeScript |
| 서버 상태 관리 | TanStack Query |
| 지도 | Leaflet (CartoDB Voyager 타일) |
| 입력 검증 | Zod (클라이언트·서버 동일 스키마) |
| AI | OpenAI API (gpt-4.1-mini) |
| 모바일 | Capacitor (iOS / Android 네이티브 래핑) |
| 배포 | Vercel |

---

## 5-2. 처리 구조

```
사용자 입력
  → [병렬] 자전거 API / 보관함 info API / 보관함 realtime API 호출
  → 반경 필터링 (Haversine)
  → 5개 전략 점수 계산 (WALK / BIKE / LOCKER_WALK×2 / LOCKER_BIKE)
  → failRisk 산정 (자원·거리·복잡도)
  → 상위 3개 전략 선정
  → [병렬] AI 판단 근거 생성 (3s timeout, 실패 시 규칙 기반 대체)
  → 결과 반환
```

이동 시간 추정 방식:
- 도보 속도: 67m/분 (약 4km/h)
- 자전거 속도: 200m/분 (약 12km/h)
- 직선 거리 × 도로 보정계수(1.3) = 추정 이동 거리

본 서비스의 이동 시간은 위경도 기반 직선 거리와 도로 보정계수를 활용한 추정값이며, 향후 지도 API 연계를 통해 실제 도로망 기반 거리로 고도화할 수 있다.

외부 API 장애 시에도 가용한 전략만으로 서비스가 정상 동작하도록 설계하여 **안정성을 확보**하였다.

# 6. 기대 효과

## 6-1. 국민 생활 개선

- 이동 판단 스트레스 감소: 여러 앱을 비교할 필요 없이 단일 서비스에서 최적 전략 제공
- 정보 탐색 시간 절감: 실시간 데이터 통합 조회로 즉시 판단 가능
- 도보 부담 감소: 짐 상황에 맞는 보관함 연계 전략 자동 추천

---

## 6-2. 행동 기반 의사결정 지원

사용자는 복잡한 비교 없이 추천된 전략을 즉시 선택하여 이동할 수 있으며, AI 판단 근거 요약을 통해 선택에 대한 신뢰도를 확보할 수 있다.

---

## 6-3. 사회적 가치

- 공공데이터 활용 활성화: 단순 조회를 넘어 의사결정 지원에 공공데이터를 활용하는 새로운 모델 제시
- 생활 밀착형 서비스: 실제 이동 상황에서 바로 사용 가능한 실용적 서비스 제공
- 이동 약자 지원 가능성: 짐이 많은 여행자, 고령자, 장거리 이동자 등 다양한 사용자층 포용

---

## 6-4. 확산 가능성

- 관광 / 출장 / 일상 등 다양한 이동 맥락에서 활용 가능
- 거점 DB 확장만으로 전국 주요 도시 확장 가능
- 공공 물류 인프라(공영킥보드, 공용주차장 등) 연계 시 서비스 범위 추가 확장 가능
